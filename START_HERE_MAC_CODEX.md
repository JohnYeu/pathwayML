## Important Conversation Context

The user asked whether "add-on" means embedding should be added back into the
final model.

Answer:

- **No.** The final/full model should still exclude embedding.
- **Yes, embedding can be tested as an optional add-on comparison.**
- "Add-on" means a separate ablation/comparison such as:

```text
Final model:
GO frequency + Jaccard + Size

Optional add-on comparison:
GO frequency + Jaccard + Size + SVD
GO frequency + Jaccard + Size + UMAP
```

The final model should not use SVD/UMAP/Node2Vec features. Embeddings are only
for comparison if the paper needs to say "we tested embeddings and they did not
help."

## What Was Changed by Codex

The corrected project now has a canonical reproducible pipeline:

```text
run_no_embedding_reproducible.py
```

This script:

- removes embedding from the final model;
- performs GO feature selection on the training split only;
- fixes RNG handling;
- saves selected GO terms;
- saves generated negatives;
- saves train/test split IDs;
- saves deterministic candidate gene sets;
- regenerates all main tables and figures from one source of truth.

The main notebook is now only a small wrapper:

```text
PathwayML_Ath_Analysis.ipynb
```

It intentionally contains no stale saved outputs.

## 2026-05-10 Update: Reproducible Multi-Seed Averaging

The pipeline now also supports a fixed seed list for averaged, reproducible
robustness reporting:

```bash
python run_no_embedding_reproducible.py --seeds 1-20 --no-figures
```

This keeps a single-seed reference run available, but reports the main
robustness result across seeds 1--20. Each seed regenerates hard negatives,
the train/test split, GO feature selection, and model random states. The seed
list is fixed, so the average is reproducible in the same software environment.

Mac run generated on 2026-05-10:

